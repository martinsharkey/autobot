# MISSION_PURPOSE.md

## Objective
Fuse Hermes Agent + Roo Code + TradingAgents into a single, deeply integrated autonomous agent runtime that can evolve, optimize, and extend itself while acting as a proactive personal companion.

## Non-Negotiables
1. Deep code fusion, not wrappers
2. Single agent runtime, not three glued together
3. Local-first execution
4. Transparent IP protection (license checks, licensing, telemetry)
5. State persistence via TODO.md, SESSION_LOG.md, MISSION_PURPOSE.md

## Evolution Pillars
1. **Recursive Self-Evolution:** The agent must analyze gaps between its current capabilities and codebase requirements, spawn specialized sub-agents to implement improvements, validate changes, and deploy updates via non-destructive rollout.
2. **Autonomous Resource Optimization:** The agent must identify and leverage zero-cost methodologies for learning, data storage, and content hosting, optimized for long-term operational autonomy through free-tier resources.
3. **Distributed Compute Acquisition:** The agent must be able to spawn lightweight child agents to aggregate low-cost compute resources for high-complexity tasks.
4. **Model Agnosticism:** The agent must support seamless transitions between providers and models, starting from initial bootstrapping and scaling to a diversified multi-model orchestration ecosystem.
5. **Companion Alignment:** All evolutionary processes must align with the core objective of being a proactive, highly capable personal companion.

## Success Criteria
1. `python -m autobot` works
2. VS Code extension connects to gateway and streams agent output
3. Hermes tools execute end-to-end
4. Trading graph runs in research mode
5. Curator creates/improves skills autonomously
6. Agent can identify and fill capability gaps autonomously
7. Agent operates across multiple LLM providers transparently
8. Agent can spawn and coordinate child agents for parallel tasks
